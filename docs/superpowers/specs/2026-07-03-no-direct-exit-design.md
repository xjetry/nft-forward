# 节点「禁止直接转发」开关

日期：2026-07-03

## 语义

每个节点（含组合）一个布尔开关 `no_direct_exit`。开启后该节点**不能作为链尾**
——即不能发起出口段直连目标。对链上任意位置生效：

- 作为入口且开启：规则必须至少级联一层线路层；
- 作为链上最后一个中间层且开启：其后必须再接一层。

校验点在 `hopsForChain`（链推导的唯一入口，管理端/用户端创建与编辑四条路径
共用）：链尾逻辑节点开启开关即拒绝。前端选择器的过滤只是便利，手工构造请求
同样被拒——这是服务端权威校验。

已有规则不受开关切换影响：开关门禁的是链推导（新建/编辑），与绑定边模式的
生效方式一致。

## 实现

- 迁移 `0037_node_no_direct_exit.sql`；`nodeCols`/`scanNode`/grants.go 三处
  scan 锁步；`POST /nodes/{id}/no-direct-exit`（管理员）。
- 前端：节点详情「节点配置」卡新增开关（单点与组合两个分支）；规则表单级联
  选择器中，某级上游开启开关则该级不显示「直接转发」选项且占位改为必选；
  链尾开启开关时红字提示并拦截提交（无可级联候选时提示死角，无法保存）。

## 测试

服务端四个场景：入口开启无 via → 400；经线路层且链尾未开启 → 200；链尾
中间层开启 → 400；关闭开关后直连恢复 → 200。
